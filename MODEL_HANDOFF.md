# MODEL_HANDOFF.md — 多模型协作切换流程

> 用途：确保在 MiniMax 和 DeepSeek 之间切换模型时，接手方能快速获取项目最新状态。
> 核心原则：模型切换不依赖聊天记忆，而依赖项目文档和 Git 记录。

---

## 切换前必读清单

**接手模型的 AI 在开始任何工作前，必须按顺序读取以下文件：**

### 第一优先级（必读）

| 序号 | 文件 | 读取原因 |
|:----:|------|----------|
| 1 | `PROJECT_BRIEF.md` | 理解项目是什么、为谁做、边界在哪 |
| 2 | `CURRENT_STATUS.md` | 了解当前版本和模块进度 |
| 3 | `AI_WORKLOG.md` | 查看上一次 AI 做了什么、遗留什么问题 |
| 4 | `TECH_DECISIONS.md` | 了解已有技术选型，避免推翻重来 |

### 第二优先级（按需读取）

| 序号 | 文件 | 何时读取 |
|:----:|------|----------|
| 5 | `TODO.md` | 需要了解下一步做什么时 |
| 6 | `ALGORITHM_SPEC.md` | 涉及测试逻辑或推荐算法时 |
| 7 | `CATALOG_STATUS.md` | 涉及专业目录或内容建设时 |
| 8 | `MAINTENANCE.md` | 执行审核/新建/修改/调参等维护操作时 |
| 9 | `REVIEW_CHECKLIST.md` | 审核专业详情内容时 |
| 10 | `PROMPT_LIBRARY.md` | 需要参考常用 Prompt 模板时 |
| 11 | `MODEL_HANDOFF.md` | 当前文件，了解协作流程 |

---

## 工作结束前必做

**当前模型的 AI 在结束工作前，必须：**

1. 更新 `AI_WORKLOG.md`，追加本次工作记录（使用模板）。
2. 更新 `CURRENT_STATUS.md`，反映最新的模块进度。
3. 如涉及专业内容变更，更新 `CATALOG_STATUS.md`。
4. 如做了技术决策，追加到 `TECH_DECISIONS.md`。
5. 确保所有代码已提交到 Git（或至少在本地有明确变更记录）。

## 模型分工建议

### MiniMax（适合的任务）

- 快速生成页面 UI 雏形和组件
- 批量生成样式代码
- 根据模板生成内容初稿
- 简单重复性任务

### DeepSeek Pro（适合的任务）

- 复杂业务逻辑（抽题算法、推荐算法）
- 架构审查和代码把关
- 项目文档撰写
- 需要深度推理的决策

### DeepSeek Flash（运行阶段）

- 仅用于线上测试报告解释生成
- 不参与开发阶段工作

---

## 项目文档与 Git 的双重保障

项目文档是**主要的事实来源**，Git 是**代码层面的事实来源**。

接手模型应：
1. 先读项目文档了解上下文。
2. 再 `git log` 查看最近的代码变更。
3. 如果文档与代码有矛盾，以代码为准，并在工作记录中标注文档需要更新。

## 初始化清单（新模型首次接入）

如果是全新的模型首次接入本项目，请执行：

```
1. 读取 PROJECT_BRIEF.md
2. 读取 CURRENT_STATUS.md
3. 读取 AI_WORKLOG.md（查看最近 3 条记录）
4. 读取 TECH_DECISIONS.md
5. 执行 ls -la 了解项目文件结构
6. 执行 git log --oneline -10 了解最近代码变更
7. 执行 git status 了解当前工作区状态
8. 开始工作
```
