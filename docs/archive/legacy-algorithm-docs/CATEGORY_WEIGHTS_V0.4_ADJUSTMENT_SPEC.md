# CATEGORY_WEIGHTS_V0.4_ADJUSTMENT_SPEC.md

## 1. 本次调整目标

本次调整不是为了让热门专业更热门，也不是为了按薪资、就业风口或社会讨论度简单推荐专业。

「专业不迷路」的推荐逻辑应服务于三个目标：

1. 帮高中生发现值得进一步了解的专业方向；
2. 帮高中生识别盲选、跟风、误解专业名带来的风险；
3. 保持公益、谨慎、非功利的推荐风格。

因此，专业权重不应直接等同于就业热度、薪资水平、社会热度或政策热度。权重只用于轻微排序，真正决定推荐结果的应是：

- 兴趣匹配；
- 学习方式适配；
- 能力接受度；
- 专业门槛；
- 院校资源要求；
- 盲选风险；
- 用户是否能接受该专业的真实学习状态。

------

## 2. 当前权重表主要问题

### 2.1 权重范围过大

当前部分专业类存在 50、60、100 等极端权重。

这会导致推荐结果被基础权重支配，而不是由用户测试答案决定。例如：

- 哲学类 100；
- 历史学类 100；
- 交叉学科类 100；
- 教育学类 60；
- 法学类 50；
- 经济学类 50。

这些权重会让某些专业在对应桶里异常容易出现，不符合“测试推荐”的逻辑。

### 2.2 用高权重弥补门类数量少

哲学类、历史学类、交叉学科类可能因为对应门类下专业类少，被赋予极高权重。

但这是不合理的。

门类下专业类少，应该通过专业目录浏览页面解决，而不是通过推荐权重放大。

推荐系统要解决的是“这个用户是否适合进一步了解该方向”，不是“这个门类是否只有一个入口”。

### 2.3 社会热度和政策热度影响过大

部分专业因为行业热、政策热、讨论热而被赋予较高权重，例如：

- 能源动力类；
- 心理学类；
- 中医学类；
- 环境科学与工程类；
- 生物工程类。

这些专业可以被关注，但不应因为热度直接高频推荐。

政策方向和行业方向是外部背景，不等于每个高中生都适合。

### 2.4 传统基础专业被压得过低

部分传统工科和基础理学专业存在权重偏低问题，例如：

- 电气类；
- 自动化类；
- 机械类；
- 材料类；
- 土木类；
- 物理学类；
- 化学类。

这些专业可能存在就业体验分化或学习难度争议，但如果用户兴趣匹配，不应被过度压低。

「专业不迷路」不是避开所有辛苦专业，而是帮助学生提前理解辛苦在哪里、适合谁、不适合谁。

## 3. 新权重分级

建议将权重范围统一压缩到 3-22。

| 等级        | 权重范围 | 含义                                                 |
| ----------- | -------- | ---------------------------------------------------- |
| 高优先      | 16-22    | 大众认知强、路径相对清晰、适配面较广，但仍需匹配兴趣 |
| 常规        | 10-15    | 兴趣匹配即可推荐                                     |
| 小众        | 6-9      | 需要较高匹配度才推荐                                 |
| 专门 / 限制 | 3-5      | 需要明确兴趣、特殊条件或资源支持才推荐               |

不建议继续使用：

- 30+
- 50+
- 60+
- 100

权重越平缓，测试结果越能体现用户真实答案，而不是被热门专业或单一门类支配。

## 5. 新增非权重字段

仅靠 `baseWeight` 不足以表达专业推荐风险。

建议为每个专业类增加以下字段：

```ts
type ResourceDependency = "low" | "medium" | "high";
type RiskLevel = "low" | "medium" | "high";
type PressureLevel = "low" | "medium" | "high";

interface CategoryWeightProfile {
  slug: string;
  name: string;
  buckets: string[];
  baseWeight: number;

  resourceDependency: ResourceDependency;
  blindChoiceRisk: RiskLevel;
  requiresSpecialCondition: boolean;
  recommendedOnlyIfHighInterest: boolean;

  longCyclePressure?: PressureLevel;
  schoolTierSensitive?: boolean;
  commonMisunderstandingRisk?: RiskLevel;

  notes?: string;
}
```

### 字段解释

#### resourceDependency

表示该专业对院校平台、实验条件、师资、企业合作、研究生培养体系的依赖程度。

建议设置为 high 的专业包括：

- 集成电路 / 微电子相关；
- 航空航天类；
- 生物医学工程类；
- 临床医学类；
- 建筑类；
- 法学类；
- 金融学类；
- 新闻传播学类；
- 心理学类。

#### blindChoiceRisk

表示高中生是否容易因为专业名字、社会热度、薪资想象而误选。

建议设置为 high 的专业包括：

- 计算机类；
- 金融学类；
- 工商管理类；
- 心理学类；
- 临床医学类；
- 集成电路 / 微电子相关；
- 新闻传播学类；
- 电子商务类；
- 环境科学与工程类；
- 生物工程类。

#### requiresSpecialCondition

表示是否存在特殊招生条件或特殊路径。

建议设置为 true 的专业包括：

- 体育学类；
- 美术学类；
- 音乐与舞蹈学类；
- 戏剧与影视学类；
- 公安学类；
- 公安技术类；

#### recommendedOnlyIfHighInterest

表示只有用户表现出明确兴趣或较强适配时，才进入推荐。

建议设置为 true 的专业包括：

#### longCyclePressure

建议设置为 high 的专业包括：

- 临床医学类；

## 6. 推荐算法调整建议

### 6.1 当前问题

如果当前算法类似：

```ts
finalScore = bucketScore * baseWeight
```

那么基础权重会过度影响结果。

例如，某个用户只是人文桶略高，也可能被 100 权重的哲学或历史强行推上前列。

### 6.2 新算法建议

建议将最终得分拆成多个部分：

```ts
finalScore =
  bucketMatchScore * 0.45 +
  learningStyleFitScore * 0.20 +
  interestSignalScore * 0.20 +
  baseWeightNormalized * 0.10 +
  riskAdjustment * 0.05;
```

其中：

- `bucketMatchScore`：用户六桶得分与专业桶的匹配度；
- `learningStyleFitScore`：学习方式适配，例如是否能接受数学、编程、背诵、实验、写作、沟通；
- `interestSignalScore`：用户对某类内容是否有明确兴趣，例如电路、生命科学、商业、语言、艺术、社会议题；
- `baseWeightNormalized`：压缩后的基础权重，只做轻微排序；
- `riskAdjustment`：根据专业风险和用户答案进行加减。

### 6.3 门槛规则

如果专业有特殊门槛，不能只靠分数推荐。

示例：

```ts
if (profile.requiresSpecialCondition && !user.specialConditionMatched) {
  reduceScoreOrMoveToCautious(profile);
}
```

如果专业要求高兴趣：

```ts
if (profile.recommendedOnlyIfHighInterest && user.interestScore < HIGH_INTEREST_THRESHOLD) {
  reduceScore(profile, strongPenalty);
}
```

如果专业院校资源依赖高：

```ts
if (profile.resourceDependency === "high") {
  addWarning("该方向对院校平台和培养资源要求较高，建议查看目标院校培养方案。");
}
```

如果专业盲选风险高：

```ts
if (profile.blindChoiceRisk === "high") {
  addWarning("该方向容易被专业名称、热度或就业想象误导，建议先了解真实学习内容。");
}
```

## 7. 推荐结果表达规范

推荐结果不要使用绝对化表达。

不要写：

- 你最适合这个专业；
- 这个专业前景最好；
- 这个专业一定好就业；
- 你应该报这个；
- 这个专业比另一个专业更值得报。

建议写：

- 你可以优先了解这个方向；
- 这个方向与你的兴趣有一定匹配；
- 但它对某些能力或条件有要求；
- 如果目标院校资源一般，需要谨慎；
- 建议继续查看培养方案、课程设置、真实学生反馈。

## 8. 模拟器验证清单

权重和算法调整后，需要通过模拟器检查以下问题：

1. 是否还有专业被极端权重支配。
2. 哲学类、历史学类、交叉学科类是否异常高频出现。
3. 计算机类、金融学类、临床医学类、法学类是否过度推荐。
4. 喜欢电学的用户是否能合理出现电子信息、集成电路、自动化、电气等方向。
5. 不喜欢编程的用户是否不会高频推荐计算机。
6. 数学接受度低的用户是否不会高频推荐强数学理工类。
7. 喜欢稳定、规则、细致的用户是否能出现会计、法学、师范、公共管理等方向，但不会被单一专业垄断。
8. 艺术、体育、公安、军工类是否必须满足特殊门槛。
9. 高资源依赖专业是否在报告里出现院校资源提醒。
10. 冷门专业是否不是永远消失，而是在高匹配时才出现。
11. 谨慎了解列表是否体现“用户期待与专业真实要求冲突”，而不是简单低分专业。
12. 推荐解释是否温和、谨慎、不替用户做决定。

## 10. 人工复核重点

自动调参和模拟器只能帮助发现明显问题，但不能替代人工判断。

人工复核时重点看：

1. 推荐是否过度迎合热门专业；
2. 冷门但适合的人是否还有机会看到；
3. 专业风险是否被充分提醒；
4. 高中生是否能看懂推荐理由；
5. 是否有“这个专业一定好 / 一定不好”的绝对化表达；
6. 是否把政策热度、行业热度误当成个人适配；
7. 是否把单个学校或单个学生的情况写成普遍规律；
8. 是否对医学、法学、集成电路、心理学、金融、新闻传播等高风险误选专业做了足够提醒。

------

## 11. v0.4 总结原则

v0.4 的核心原则可以概括为：

> 权重只做轻微排序，适配才是核心；
> 热门不能自动高推，冷门不能永久消失；
> 高风险专业必须提醒，特殊专业必须设门槛；
> 网站不替学生做决定，只帮学生看懂方向、发现风险、继续核实。
