# ALGORITHM_SPEC.md — 测试推荐算法规格说明

> 用途：记录方向测试的算法设计规格，供开发和 AI 参考。
> 版本：v0.3 — 六桶粗筛 + 自适应方向抽题 + 三层推荐输出
>
> ⚠️ 本文件只描述算法框架和参数，**不包含具体测试题和算法代码**。
> 题目仓库和代码实现另见对应源文件。

---

## 1. 算法目标

### 1.1 算法定位

**名称**：专业不迷路 · 自适应题库抽题推荐算法 v0.3

**对外名称**：专业倾向小测试（体验类似 MBTI 的答题流程，但**不是心理测评**）

**一句话说明**：
先用 6 桶通用题粗判用户方向倾向，再从匹配方向中抽题确认细分兴趣，
最后输出「门类 → 专业类 → 具体专业」三层推荐，附避坑提醒。

### 1.2 核心原则

| 原则 | 说明 |
|------|------|
| 不替用户做决定 | 只帮助缩小范围，不输出"最适合你" |
| 不问无关问题 | 文科用户不受工科题打扰，反之亦然 |
| 避坑不恐吓 | 提醒风险但不制造焦虑，用"建议先看清楚"而非"千万别选" |
| AI 只解释不打分 | 规则系统完成所有评分和推荐，AI 仅负责把结果说成人话 |
| 结果可追溯 | 每个推荐方向都有对应的得分和触发规则，不是黑箱 |

### 1.3 定位红线

- ❌ 不输出"你最适合某某专业"
- ❌ 不做分数、位次、院校、录取概率判断
- ❌ 不替代正式志愿填报决策
- ❌ 不将用户原始答案直接丢给 AI 做推荐
- ❌ 不用"精准决定人生""一键选出最优专业"等焦虑文案

## 2. 当前方向桶（6 桶粗筛）

### 2.1 六桶定义

```
                          测试入口
                             │
                    8 道通用粗筛题
                             │
          ┌──────┬──────┬────┼────┬──────┬──────┐
          ▼      ▼      ▼    │    ▼      ▼      ▼
         B1     B2     B3   B4   B5     B6
        人文   社科   商科  理工  生命  艺术
                                      健康  创作
```

| 编号 | 方向桶 | 覆盖学科门类 | 覆盖专业类（示例） |
|:----:|--------|-------------|-------------------|
| B1 | 人文 | 哲学、文学、历史学 | 中国语言文学类、外国语言文学类、新闻传播学类、历史学类 |
| B2 | 社科 | 法学、教育学 | 法学类、政治学类、社会学类、教育学类、马克思主义理论类 |
| B3 | 商科 | 经济学、管理学 | 经济学类、金融学类、工商管理类、公共管理类、电子商务类 |
| B4 | 理工 | 理学、工学 | 数学类、物理学类、计算机类、电子信息类、机械类、土木类等 |
| B5 | 生命健康 | 医学、农学 | 临床医学类、药学类、护理学类、植物生产类、动物医学类 |
| B6 | 艺术创作 | 艺术学 | 美术学类、设计学类、音乐与舞蹈学类、戏剧与影视学类 |

### 2.2 桶内细分

粗筛结束后，根据命中的桶进入细分方向。每个桶内保留细分方向（参考旧版 8 通道，归入六桶）：

| 桶 | 细分方向 |
|----|---------|
| B1 人文 | 语言文学 / 历史哲学 / 新闻传播 |
| B2 社科 | 法学方向 / 教育与社会 / 公共管理 |
| B3 商科 | 经济理论 / 金融财会 / 工商管理 |
| B4 理工 | 理学基础 / 计算机与信息 / 工程与制造 |
| B5 生命健康 | 临床医学 / 药学与健康 / 农学与生态 |
| B6 艺术创作 | 美术与设计 / 音乐舞蹈 / 戏剧影视 |

### 2.3 相邻桶定义

只有**相邻桶**之间才互相抽取校验题。非相邻桶不交叉——这是文科保护规则的核心。

```
B1 人文 ←→ B2 社科 ←→ B3 商科 ←→ B4 理工 ←→ B5 生命健康
   ↕                        ↕              ↕
B6 艺术创作                  B6           B6
```

| 主方向桶 | 可校验的相邻桶 | 禁止深入的桶 |
|----------|---------------|-------------|
| B1 人文 | B2 社科、B6 艺术 | B4 理工、B5 生命健康 |
| B2 社科 | B1 人文、B3 商科 | B4 理工深题 |
| B3 商科 | B2 社科、B4 理工 | B5 生命健康深题、B6 艺术深题 |
| B4 理工 | B3 商科（数据方向）、B5 生命健康（生物医学工程） | B1 人文深题、B6 艺术深题 |
| B5 生命健康 | B4 理工（生物、化学基础）、B2 社科（公共卫生） | B1 人文深题、B3 商科深题 |
| B6 艺术创作 | B1 人文、B4 理工（数字媒体、工业设计） | B5 生命健康深题 |

## 3. 通用题数量建议

### 3.1 粗筛通用题

| 参数 | 建议值 | 说明 |
|------|:------:|------|
| 通用粗筛题数 | **8 题** | 覆盖全部 6 桶，每桶至少 1 题可直接为其加分 |
| 题型分布 | 学科偏好 2 + 任务场景 2 + 人与事倾向 1 + 抽象具体偏好 1 + 职业期待 1 + 明显排斥 1 | 不涉及专业术语，高中生都能理解 |
| 每道题选项数 | 4-5 个 | 每个选项映射到不同桶的加分 |
| 通用题最低库存 | **20 题** | 保证轮换，避免刷题记忆 |

### 3.2 通用题设计原则

- 不出现任何专业名称（"计算机""法学""医学"等字眼不出现在题干和选项中）
- 从**日常场景**出发（如"你喜欢以下哪种周末活动"）
- 选项之间不暗示优劣——"喜欢读书"和"喜欢动手"只是不同，没有高下
- 至少 1 道题能触发明显排斥信号（如"以下哪种最让你头疼"）

## 4. 分流题数量建议

### 4.1 粗筛后分流向题

粗筛结束后，根据命中桶进入**自适应分流阶段**：

| 用户类型 | 命中条件 | 主桶题数 | 相邻桶题数 |
|----------|----------|:------:|:--------:|
| 单桶明确 | 某桶得分 ≥ 62 且领先第二名 ≥ 8 分 | **6 题** | **2 题**（仅相邻桶） |
| 双桶均衡 | 前两名都 ≥ 55 且差距 < 8 分 | **各 4 题** | — |
| 探索型 | 最高分 < 58 或前三名接近 | **前三桶各 3 题** | — |

### 4.2 每桶方向题库最低库存

| 桶 | 最少题数 | 第一版优先 |
|----|:------:|:----------:|
| B1 人文 | 10 | ✅ |
| B2 社科 | 10 | ✅ |
| B3 商科 | 10 | ✅ |
| B4 理工 | 12 | ✅（桶最大，需更多题覆盖细分方向） |
| B5 生命健康 | 8 | 次优先 |
| B6 艺术创作 | 8 | 次优先 |
| **合计方向题** | **58 题** | 第一版目标 50-60 题 |

### 4.3 文科保护规则（硬规则）

触发条件：B1 人文或 B2 社科进入前 2 **且** B4 理工 < 55

效果：
- B4 理工方向题数量 = **0**（不问任何工科深题）
- 以下专业类不进入优先推荐：**计算机类、电子信息类、自动化类、电气类、机械类、土木类**
- 用户未主动表达 B4 兴趣时，也不放入"可以了解"区

逆规则：用户在第 0 步明确选了物理、或主动勾选"想了解计算机/电子信息"，则文科保护规则**不生效**。

## 5. 跨方向校验题数量建议

### 5.1 校验题机制

校验题仅在**相邻桶**之间抽取（见 §2.3 相邻桶定义表），用于：
- 确认用户对相邻方向的真实兴趣（而非误触）
- 发现用户对自己认知的偏差（以为自己喜欢 A，实际上更适合 B）

| 参数 | 建议值 |
|------|:------:|
| 校验题总数 | **1-2 题** |
| 抽取来源 | 仅从主桶的相邻桶中抽取 |
| 校验题触发条件 | 相邻桶方向得分 ≥ 50（即并非完全无关，有微弱信号） |

### 5.2 禁止事项

- ❌ 绝不从非相邻桶（禁止深入桶）抽取校验题
- ❌ 校验题不是"硬塞不相关的专业"
- ❌ 如果所有相邻桶得分都 < 50，校验题数量 = 0

## 6. 当前阈值建议

### 6.1 桶得分阈值

| 参数 | 当前值 | 可调范围 | 含义 |
|------|:------:|:--------:|------|
| 高分阈值（HIGH） | **62** | 58-68 | 命中此分数以上，视为强信号 |
| 有效阈值（MID） | **55** | 50-60 | 命中此分数以上，进入细化抽题 |
| 微弱信号（LOW） | **50** | 45-52 | 仅触发校验题，不进入主方向抽题 |
| 排除阈值 | **45** | 40-50 | 低于此分数的桶不再出现 |
| 单桶领先差 | **8** | 6-12 | 最高桶需领先第二多少才算"单桶明确" |
| 探索型上限 | **58** | 55-62 | 最高分低于此时判为探索型 |

### 6.2 得分计算公式

```
桶总分 = 通用题得分（标准化至 0-100）
       + 方向题得分（标准化至 0-100）
       - 排斥扣分（每触发一次排斥选项 -10）
       + 第 0 步主动兴趣加分（+5 /每项，最多 +10）

最终归一化到 0-100
```

**通用题和方向题权重**：通用题权重 0.4，方向题权重 0.6（方向题是细化确认，权重更高）。

### 6.3 选科背景惩罚

| 情况 | 惩罚值 |
|------|:------:|
| B4 理工方向 + 未选物理 | **-30 分** |
| B4 理工方向 + 未选化学（材料、化工相关） | -15 分 |
| B5 生命健康方向 + 未选生物 | -20 分 |
| B5 生命健康方向 + 未选化学（药学、临床相关） | -20 分 |

选科惩罚**不影响测试流程中的出题**，仅在最终得分计算时生效。

## 7. 推荐结果分组

### 7.1 三层输出结构

```
结果页
├── 📌 你的方向画像（简短文字描述 + 桶得分可视化）
│
├── 🎯 优先了解（3-5 个专业类）
│   ├── 专业类 A → 具体专业详情页入口
│   ├── 专业类 B → 具体专业详情页入口
│   └── ...
│
├── 👀 可以了解（2-4 个专业类）
│   ├── 专业类 C → 具体专业详情页入口
│   └── ...
│
├── ⚠️ 谨慎了解（2-4 个专业类，含避坑标签）
│   ├── 专业类 D → 先看详情页再决定
│   └── ...
│
└── 💡 下一步建议（温和的行动引导）
```

### 7.2 各分组的生成逻辑

| 分组 | 来源 | 排序依据 |
|------|------|----------|
| 🎯 优先了解 | 主桶 Top 1-2 专业类 + 维度高度匹配 | 桶得分 × 0.3 + 维度匹配 × 0.45 + 主动兴趣 × 0.1 |
| 👀 可以了解 | 相邻桶 Top 1 专业类 + 主桶剩余匹配类 | 同上，但阈值降低 10 分 |
| ⚠️ 谨慎了解 | 桶得分够但触发避坑规则 / 选科不匹配 / 名字误解 | 带标签展示，不按分数排序 |

### 7.3 输出数量约束

| 区域 | 最小 | 最大 |
|------|:----:|:----:|
| 优先了解 | 1 | 5 |
| 可以了解 | 0 | 4 |
| 谨慎了解 | 0 | 4 |

探索型用户：优先了解 ≤ 2 个，可以了解可放宽至 6 个。

## 8. 避坑标签

### 8.1 三大避坑类别

#### 🔥 热门跟风风险

| 标签 | 触发条件 | 处理 |
|------|----------|------|
| `热门跟风` | 用户选择理由含"热门/高薪/他人推荐/家长要求" | 保留推荐方向，但加"建议先了解真实学习内容再做判断" |
| `薪资预期偏差` | 用户仅因高薪关注某方向 | 触发详情页的"常见误解"模块高亮 |
| `信息茧房` | 用户仅了解 1-2 个热门专业类 | 在"可以了解"区推荐相邻但不同的方向 |

#### 🚫 学习方式排斥

| 标签 | 触发条件 | 处理 |
|------|----------|------|
| `长周期高投入` | 用户高分命中临床医学但排斥长期投入 | 医学类降入"谨慎了解"，附提醒"医学本科 5 年+规培 3 年，学业周期比大多数专业长" |
| `大量阅读写作` | 用户排斥大量阅读但命中法学/文学 | 法学/文学类降入"谨慎了解"或"可以了解" |
| `规则与重复` | 用户排斥规则、细节但命中会计/法学 | 触类降入"谨慎了解"，附提醒"这些专业需要较细致地处理规则和文本" |
| `动手与实验` | 用户排斥动手操作但命中理工 | B4 理工方向降一档 |

#### 🤔 方向误解

| 标签 | 触发条件 | 处理 |
|------|----------|------|
| `名字误解` | 用户对专业类名称存在明显认知偏差（如工商管理=毕业当管理层） | 推荐结果中直接引导至详情页的"常见误解"模块 |
| `表面兴趣` | 用户表达兴趣的理由与该专业类实际学习内容严重不符 | 降入"谨慎了解"，提醒"建议先看看这个专业类到底学什么再决定" |
| `文理认知偏差` | 用户选科与方向严重不匹配（未选物理但关注电子信息） | 不进入优先推荐区 |

### 8.2 避坑标签展示规则

- 每个"谨慎了解"区的专业类**最多 2 个标签**
- 标签**展示在专业类名称下方**，小字体灰色底
- 标签文案为**建议行动**而非否定判断：
  - ✅ "建议先了解真实学习内容"
  - ❌ "你不适合这个专业"

## 9. 后续调参方法

### 9.1 调参原则

调参**不靠感觉**，靠匿名反馈数据。每轮内测后收集以下指标：

| 指标 | 收集方式 | 目标值 |
|------|----------|:------:|
| 结果有用率 | 用户评分 1-5，≥4 算"有用" | ≥ 70% |
| 明显跑偏率 | 用户标记"这个方向完全不相关" | ≤ 15% |
| 无关题出现率 | 用户标记"这道题跟我没什么关系" | ≤ 10% |
| 推荐点进率 | 优先了解区的专业类被点击打开 | ≥ 50% |
| 谨慎提醒有帮助 | 用户标记"这个提醒对了我有帮助" | 不做硬目标，仅观测 |

### 9.2 关键调参旋钮

| 参数 | 调大效果 | 调小效果 |
|------|----------|----------|
| 高分阈值（HIGH） | 需要更强信号才触发"优先"，减少推荐数 | 更容易进入优先推荐 |
| 有效阈值（MID） | 减少进入细化抽题的桶数 | 更多桶进入细化抽题 |
| 单桶领先差 | 更多用户归入双桶或探索型 | 更多用户归入单桶明确 |
| 通用题权重 | 粗筛影响更大 | 方向题细化影响更大 |
| 选科背景惩罚 | 选科不匹配时推荐更保守 | 选科约束更宽松 |

### 9.3 调参节奏

1. **内测阶段**（≤ 50 人）：先调阈值（HIGH/MID/领先差），不调权重
2. **公测阶段**（≥ 200 人）：根据反馈调权重和选科惩罚
3. **稳定阶段**：参数锁定，只补充题库和内容

## 10. 测试全流程速览

```
┌─────────────────────────────────────────────┐
│  第 0 步：基础信息（2 题）                     │
│  阶段（高一/高二/高三）+ 选科 + 已有兴趣        │
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│  第 1 步：8 道通用粗筛题                      │
│  覆盖 6 桶，无专业术语，场景化                  │
│  → 计算 6 个桶的初始得分                       │
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│  判型：单桶 / 双桶 / 探索型                    │
│  + 文科保护规则检查                            │
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│  第 2 步：自适应分流题                         │
│  主桶 6 题 + 相邻桶 2 题 + 避坑 1 题 + 校验 1 │
│  → 计算桶得分 + 维度匹配 + 避坑标签             │
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│  计算最终得分 + 应用选科惩罚 + 避坑修正         │
│  → 生成三层推荐（优先 / 可以 / 谨慎）           │
└──────────────┬──────────────────────────────┘
               ▼
┌─────────────────────────────────────────────┐
│  AI 解释（仅接收结果包，不参与打分）             │
│  → 呈现推荐 + 避坑提醒 + 下一步建议             │
└─────────────────────────────────────────────┘
```

## 11. AI 使用边界

| 环节 | AI 参与？ | 说明 |
|------|:--------:|------|
| 出题 | ❌ | 题目预先设计，AI 不生成题目 |
| 打分 | ❌ | 规则系统完成，AI 不参与任何评分 |
| 排序 | ❌ | 规则排序，AI 不调整推荐顺序 |
| 解释结果 | ✅ | AI 将结果包翻译成高中生能看懂的大白话 |
| 新增推荐方向 | ❌ | AI 不得新增规则系统未给出的专业类 |
| 预测录取 | ❌ | 绝对禁止 |
| 替用户做决定 | ❌ | 绝对禁止 |

**AI 输入格式**（示例）：规则系统输出的**结果包**（方向画像 + 三层推荐列表 + 避坑标签 + 用户补充文字），AI 只解释、不创造。
