# REVIEW_CHECKLIST.md — 专业详情内容审核清单

> 用途：新增或修改专业详情内容后的审核流程。
> 原则：内容先可信再好看。宁缺毋滥，宁可标 draft 也不发未经确认的内容。

---

## 审核流程

```
作者提交 → 初审（事实检查）→ 复审（表达检查）→ 上线（reviewStatus: 'reviewed'）
```

| 步骤 | 谁做 | 做什么 |
|:----:|------|--------|
| 1. 提交 | 共建者/AI | 写入 `src/data/major-details.ts`，标记 `reviewStatus: 'draft'` |
| 2. 初审 | 项目维护者 | 逐条核实事实（课程设置、学习场景、就业方向），对照至少 2 个高校培养方案 |
| 3. 复审 | 项目维护者 | 检查表达是否说大白话、有无夸大或贬低、有无制造焦虑 |
| 4. 上线 | 项目维护者 | 改 `reviewStatus: 'draft'` → `'reviewed'`，`npm run build` |

---

## 初审清单（事实检查）

逐条回答以下问题，全部通过才算初审完成：

### 1. 一句话解释
- [ ] 高中生能一句话看懂这个专业是做什么的吗？
- [ ] 有没有过度简化导致理解偏差？

### 2. 大学学什么
- [ ] 所列课程确实是该专业的核心课程吗？
- [ ] 课程名称用的是通用名称（不是某个学校特有的）？
- [ ] 课程数量适可而止（7-10 条），不是堆砌培养方案？

### 3. 与高中不同
- [ ] 对比的是普通高中生能感知的差异？
- [ ] 没有暗示高中学习"没用"？

### 4. 适合谁 / 5. 不适合谁
- [ ] 有没有给某一类人贴标签？
- [ ] "不适合"的表述是不是温和的提醒而非否定？
- [ ] 有没有歧视性表述（性别、地域、家庭背景等）？

### 6. 常见误解
- [ ] 每条误解确实是常见误解（不是冷门或杜撰的）？
- [ ] 纠正的表述是否有依据？
- [ ] 有没有一竿子打死（如"千万别信"）？

### 7. 学习场景
- [ ] 描述的场景是真实的（不美化也不妖魔化）？
- [ ] 课堂、课外、期末三个维度都有覆盖？

### 8. 避坑提醒
- [ ] 是不是温和的"建议看清楚"而非"千万别选"？
- [ ] 是否提供了具体的行动建议（如"去官网看培养方案"）？

---

## 复审清单（表达检查）

- [ ] 全文没有出现"你最适合""强烈推荐""千万别选"等替人决定的句式
- [ ] 没有出现"毕业就能进大厂""薪资很高""前途无量"等就业承诺
- [ ] 没有出现对任何专业的贬低（"这个专业没用""学了等于白学"）
- [ ] 全文用大白话（假设给一个高二学生看，他能看懂 90%）
- [ ] 没有堆砌官方术语（培养目标、课程体系、人才规格…）
- [ ] 分段合理，每段不超过 200 字
