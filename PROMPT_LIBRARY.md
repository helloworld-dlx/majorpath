# PROMPT_LIBRARY.md — 常用 Prompt 模板库

> 用途：收集和整理项目开发中常用的 Prompt 模板，提高 AI 协作效率。
> 维护规则：每次发现好用的 Prompt 时追加，标注适用场景和模型。

---

## 1. 项目脚手架搭建

```
在当前目录初始化一个 Astro 项目，要求：
- 使用 TypeScript
- 集成 Tailwind CSS
- 集成 React 组件支持
- 使用默认的博客/内容站模板

## 2. 页面与组件开发

### 创建基础布局组件

```
为 Astro 项目创建基础布局组件：
- Header：包含网站名称"专业不迷路"、导航链接（首页、专业目录、方向测试、关于）
- Footer：包含公益属性声明、免责链接、共建入口
- Layout.astro：包裹 Header 和 Footer 的基础布局
- 使用 Tailwind CSS 样式
- 设计风格：简洁、温暖、学生友好，非商业化
```

适用模型：MiniMax（快速出页面）/ DeepSeek Pro（审查）

### 创建首页

```
实现首页，要求：
- 顶部：项目一句话介绍"用说大白话的方式，帮你看懂大学专业"
- 中部：两个等大的入口卡片
  - 左卡片"先看专业目录"（适合已有目标的用户）
  - 右卡片"我没方向，做个小测试"（适合没方向的用户）

适用模型：MiniMax

## 3. 数据结构

### 本科专业目录数据

```
创建本科专业目录的 TypeScript 数据结构，要求：
- 三级结构：学科门类 → 专业类 → 具体专业

- 将计算机类、工商管理类标记为优先建设
```

适用模型：DeepSeek Pro

## 4. 算法实现

### 方向通道分数计算

```
实现方向通道分数计算函数，要求：
- 输入：用户 8 道通用题的答案（每题选项对应各通道的加减分）
- 输入：用户选科背景（物理/历史等）
- 输入：用户主动兴趣方向
- 输出：8 个方向通道的 0-100 分数
- 参数可配置（阈值、权重）
- 纯函数，无副作用
- 使用 TypeScript
```

适用模型：DeepSeek Pro

### 自适应抽题

```
实现自适应抽题函数，要求：
- 输入：用户类型（单主线/双主线/探索型）
- 输入：方向通道分数
- 输入：题库数组
- 根据用户类型和主方向，从题库中抽取对应数量的题目
- 遵循以下规则：
  - 单主线：主方向 6 题 + 相邻方向 2 题 + 避坑 1 题 + 校验 1 题
  - 双主线：第一方向 4 题 + 第二方向 4 题 + 避坑 1 题 + 校验 1 题
  - 探索型：前三方向各 3 题 + 避坑 1 题 + 可能追加 2 题
- 实现文科倾向保护：未选物理 + 工程通道低分时不抽工科深题
- 使用 TypeScript
```

适用模型：DeepSeek Pro

## 5. 内容生成

### 专业详情页内容初稿

```
为"计算机类"专业撰写详情页内容初稿，面向高中生，要求：
- 一句话解释：用高中生能懂的话
- 大学主要学什么：列出核心学习内容，避免堆术语
- 和高中学科有什么不同：重点讲高中信息技术/数学与大学计算机的差异
- 适合什么样的人：说明学习方式、兴趣、能力倾向
- 不太适合什么样的人：温和提醒
- 常见误解：澄清"学计算机=修电脑""会打游戏=适合学计算机"等
- 真实学习场景：用一小段描述计算机专业学生的一天
- 未来可能方向：简要说明，不承诺薪资
- 字数：每部分 80-150 字
- 语气：像学长学姐在聊天，不像官方招生简章
```

适用模型：MiniMax（生成初稿）/ DeepSeek Pro（优化）

## 6. AI 报告生成

### 测试报告 System Prompt

```
你是"专业不迷路"的方向解释助手。你的任务是把测试结果用高中生听得懂的话解释出来。

规则：
1. 你只根据我给你的结果包做解释，不要新增任何结果包里没有的专业类。
2. 不要说"你应该选某某专业"，说"值得先了解一下某某方向"。
3. 不要预测录取概率或就业薪资。
4. 谨慎方向要用温和的语气，说成"建议先看清楚再判断"，不要直接否定。
5. 最后必须加上"这个测试只是帮你缩小了解范围，最终选择还要结合分数、院校、家庭情况和自己的想法"。
6. 总字数控制在 300-500 字。

结果包格式：
- 方向画像：[一句话描述]
- 优先了解专业类：[列表]
- 可以了解：[列表]
- 谨慎了解：[列表]
- 避坑提醒：[列表]
- 用户补充：[文字]
```

适用模型：DeepSeek Flash（运行阶段）

## 模板使用说明

- 所有模板均需根据实际情况调整细节。
- 标有"适用模型"的是建议，可根据手头可用的模型灵活选择。
- 使用 Prompt 后如效果好，可在模板后追加"已验证"标记。
- 使用 Prompt 后如效果不好，记录问题和改进版。
